// 01   Project Background

Detecting threats before they escalate

NetIDS was built to address a critical gap in accessible network security tooling. Most IDS solutions are heavyweight, expensive, or require enterprise infrastructure. NetIDS provides serious detection capabilities — in a tool anyone can run from the command line.

The system operates in two modes: live packet sniffing via Scapy for real-time monitoring, and PCAP file analysis for forensic investigation. Seven independent detection engines run in parallel, each using configurable rolling time-window thresholds with per-source IP tracking. Alerts escalate automatically — a source crossing MEDIUM then HIGH thresholds generates distinct escalation alerts.

Alerts are enriched with IP geolocation and ASN data, mapped to MITRE ATT&CK techniques, and correlated into incidents with 0–100 risk scores and attack chain detection. Every incident can be triaged instantly by Claude Haiku AI, which classifies the attack, explains the attacker's likely goal, and recommends SOC response actions.

  • TCP
    TCP Port Scan Detection Tracks distinct destination ports per source IP within a 15s sliding window. Escalates LOW → MEDIUM → HIGH at 15 / 30 / 60 ports. MITRE T1046.
  • SYN
    SYN Burst / SYN Flood Detection Detects SYN-only packets (no ACK) per source in an 8s window — hallmark of DoS and connection exhaustion attacks. Thresholds: 8 / 12 / 50 SYNs. MITRE T1498.
  • ICM
    ICMP Sweep Detection Monitors ICMP echo requests per source in a 30s window to detect host discovery and reconnaissance sweeps. Thresholds: 5 / 10 / 30 packets. MITRE T1018.
  • LAT
    Lateral Movement Detection Tracks SSH SYN probes from one source to many unique destination hosts in a 60s window. Flags internal network traversal attempts. Thresholds: 4 / 8 / 20 hosts. MITRE T1021.
  • DNS
    DNS Tunneling Detection Dual-signal: high query rate (20 / 40 / 80 in 10s) and oversized query labels (≥75 chars) indicating base64-encoded exfiltration payloads. MITRE T1071.004.
  • WEB
    Web Exploit Detection Pattern-matches SQL injection, path traversal, and command injection payloads in HTTP traffic. Fires HIGH on SQLi / command injection, MEDIUM on traversal. MITRE T1190.
  • SLW
    Slow Loris Detection Counts concurrent half-open TCP connections (SYN with no FIN/RST) targeting web server ports. Connections expire after 120s. Thresholds: 10 / 20 / 40. MITRE T1499.
  • COR
    Incident Correlation & Attack Chain Detection Groups alerts by source IP in a 120s window into scored incidents (0–100 risk). Recognizes 7 multi-stage campaigns — Host Discovery → Port Scan, Recon → Lateral Movement, Web Exploit → Data Exfiltration, and more.
  • AI
    Claude Haiku AI Triage Every incident can be analyzed on demand by Claude Haiku. Returns a structured attack classification, 2–3 sentence analysis of attacker intent, and concrete SOC response steps. Results cached in triage.jsonl.
  • ENR
    IP Enrichment & MITRE Mapping Every alert enriched with geolocation, ASN, and reverse DNS. Geo coordinates power the live attack map. All 7 alert types mapped to MITRE ATT&CK technique IDs automatically.
  • UI
    SOC Web Dashboard Authenticated Flask UI (PBKDF2-SHA256, rate-limited login). Live alert feed, geo attack map, incident cards with inline AI triage, severity charts, alerts-per-minute counter, and one-click demo data loading.

What was achieved

  • Real-time packet inspection via Scapy with dual live/PCAP modes
  • Severity escalation — LOW → MEDIUM → HIGH fires as attack intensity grows, with per-severity independent cooldowns
  • 7 multi-stage attack chain patterns recognized from correlated incidents (e.g. Host Discovery → Port Scan → Lateral Movement)
  • Claude Haiku AI triage: on-demand attack classification, analyst explanation, and SOC response steps per incident
  • GeoIP enrichment + Leaflet.js live attack map pinpointing source countries and ASNs
  • PBKDF2-SHA256 authentication with login rate limiting and persistent session keys
  • One-click demo seeding with 23 realistic multi-country alerts covering all 7 attack types and all 3 severity levels
  • Audio alert tones (Web Audio API) on new HIGH/MEDIUM detections — no browser plugins required

Future work

  • Integrate the trained Isolation Forest model for ML-based anomaly scoring alongside rule-based detectors
  • HTTPS / TLS termination for the dashboard in production deployments
  • Persistent storage — replace flat JSONL with a database for large-scale deployments
  • Additional detectors: UDP flood, ARP poisoning, credential stuffing
  • Unit test suite covering each detection rule's threshold and escalation behavior
  • SIEM integration — forward alerts to Elasticsearch or Splunk via the existing JSON Lines format
