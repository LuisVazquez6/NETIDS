NetIDS

A modular, Python-based Network Intrusion Detection System that detects 7 attack types in real time — from port scans and SYN floods to DNS tunneling and Slow Loris. Features incident correlation with risk scoring, MITRE ATT&CK mapping, IP geolocation enrichment, and a Flask web dashboard.

// 01   Project Background

Detecting threats before they escalate

NetIDS was built to address a critical gap in accessible network security tooling. Most IDS solutions are heavyweight, expensive, or require enterprise infrastructure. NetIDS provides serious detection capabilities — in a tool anyone can run from the command line.

The system operates in two modes: live packet sniffing via Scapy for real-time monitoring, and PCAP file analysis for forensic investigation. Seven independent detection engines run in parallel, each using configurable rolling time-window thresholds with per-source IP tracking.

Alerts are automatically enriched with IP geolocation and ASN data, mapped to MITRE ATT&CK techniques, and correlated into incidents with 0–100 risk scores. Output is JSON Lines — structured, machine-readable, and ready for any SIEM or alerting pipeline.

  • TCP
    TCP Port Scan Detection Tracks distinct destination ports per source IP within a 15s window. Fires at 15 / 30 / 60 ports (LOW / MEDIUM / HIGH). MITRE T1046.
  • SSH
    SSH Brute Force Detection Counts SYN-only packets to port 22 per source within a 30s window. Thresholds: 8 / 20 / 50 attempts. MITRE T1110.
  • SYN
    SYN Burst / SYN Flood Detection Detects SYN-only packets (no ACK) per source within an 8s window — a hallmark of DoS attempts. MITRE T1499.
  • ICM
    ICMP Flood Detection Monitors ICMP echo requests per source within a 10s window. Catches both flood attacks and reconnaissance ping sweeps. MITRE T1498.
  • DNS
    DNS Tunneling Detection Dual-signal: high query rate (20 / 40 / 80 in 10s) AND long query names (≥75 chars). Both signals together escalate severity to HIGH. MITRE T1071.004.
  • HTT
    HTTP Brute Force Detection Tracks POST requests per (source, destination, port) tuple on ports 80 / 443 / 8080 within a 20s window. MITRE T1110.003.
  • SLW
    Slow Loris Detection Counts concurrent half-open TCP connections (SYN seen, no FIN/RST). Connections expire after 120s. Thresholds: 10 / 20 / 40. MITRE T1499.
  • COR
    Incident Correlation & Risk Scoring Groups alerts by source IP within a 120s window into scored incidents (0–100). Detects multi-stage attack chains — Recon → SSH, Recon → Flood, Recon → Web Attack.
  • ENR
    IP Enrichment & MITRE Mapping Every alert enriched with geolocation, ASN, and reverse DNS via ip-api.com. Automatically mapped to the corresponding MITRE ATT&CK technique ID.
  • UI
    Flask Web Dashboard Authenticated web UI (PBKDF2-SHA256, session-based, rate-limited). Live alert and incident tables, filter by severity / type / source IP, Plotly charts.
